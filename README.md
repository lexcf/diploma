# Детектор аномалий сетевого трафика

Программа для обучения модели без учителя на сетевом трафике и детекции аномалий в реальном времени. При обнаружении аномалии формируется архив с логом трафика за последние N минут и отправляется на веб-сервер.

## Описание

Программа работает в **двух режимах**:

1. **Обучение** — захват трафика в течение заданного времени, агрегация по временным окнам, обучение модели Isolation Forest и сохранение модели в файл.
2. **Детекция** — непрерывный захват трафика в реальном времени, агрегация по тем же временным окнам, предсказание аномалий; при срабатывании выводится информация в консоль и (опционально) на веб-сервер отправляется ZIP-архив с логом трафика за последние N минут.

Данные агрегируются **только по временному окну** фиксированной длины (в секундах). Размер окна и чувствительность детектора задаются в конфигурационном файле или через аргументы командной строки.

## Как это работает

### Общая схема

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         РЕЖИМ ОБУЧЕНИЯ (train)                          │
├─────────────────────────────────────────────────────────────────────────┤
│  Сетевой интерфейс → Захват пакетов → Извлечение признаков →            │
│  → Агрегация по окнам → Обучение Isolation Forest → Сохранение модели   │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                        РЕЖИМ ДЕТЕКЦИИ (detect)                          │
├─────────────────────────────────────────────────────────────────────────┤
│  Сетевой интерфейс → Захват пакетов → Извлечение признаков              │
│       ↓                      ↓                                            │
│  TrafficLogger     TimeWindowAggregator → Модель → Аномалия?             │
│  (буфер N минут)         (окна)              ↓                           │
│       ↓                    ↓              Да → Вывод в консоль +         │
│       └────────────────────┴──────────────→ ZIP(трафик за N мин) →      │
│                                    POST на веб-сервер (если включено)    │
└─────────────────────────────────────────────────────────────────────────┘
```

### Режим обучения (train)

1. Запускается захват пакетов с указанного интерфейса на заданное время (минуты).
2. Из каждого пакета извлекаются признаки (IP, порты, протокол, длина, TTL и т.д.).
3. Пакеты группируются в **временные окна** фиксированной длины (например, 5 секунд). Для каждого окна считаются агрегированные метрики: число пакетов, пакетов/сек, уникальные IP/порты, статистики по протоколам и длине пакетов.
4. По этим метрикам обучается **Isolation Forest** (без учителя). Модель и нормализатор (StandardScaler) сохраняются в файл (например, `anomaly_model.pkl`).

### Режим детекции (detect)

1. Загружается сохранённая модель и (опционально) порог по `anomaly_score`.
2. Запускается непрерывный захват пакетов. Каждый пакет:
   - добавляется в **TrafficLogger** — кольцевой буфер, хранящий трафик за последние N минут (для архива при аномалии);
   - передаётся в **TimeWindowAggregator** — по мере заполнения окон получаются агрегированные векторы.
3. Для каждого завершённого окна модель выдаёт предсказание (норма/аномалия) и значение `anomaly_score`.
4. Если зафиксирована аномалия:
   - в консоль выводится подробная информация по окну;
   - при включённой настройке формируется ZIP-архив с JSONL-логом трафика за последние N минут и отправляется POST-запросом на веб-сервер (URL задаётся в конфиге).

### Демонстрационный веб-сервер (alert_server.py)

Простой Flask-сервер принимает POST-запросы на `/anomaly` с полями:

- `message` — текстовое описание аномалии;
- `file` — ZIP-архив с логом трафика.

Сервер сохраняет архив в каталог `uploads/` и выводит в консоль сообщение о полученной аномалии и путь к файлу. Этого достаточно для демонстрации приёма алертов.

### Требования

- Python 3.7+
- Права root (sudo) для захвата сетевых пакетов

Из каталога с исходным кодом:

```bash
pip install -r requirements.txt
```

## Конфигурация

Параметры вынесены в файл **`config.py`**. Их можно менять прямо в файле; значения из конфига используются как умолчания для CLI.

### Блок `training` (режим обучения)

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `interface` | Сетевой интерфейс | `"eth0"` |
| `duration_minutes` | Длительность захвата в минутах | `5` |
| `window_size_seconds` | Размер временного окна (сек) | `5.0` |
| `model_path` | Путь к файлу модели | `"anomaly_model.pkl"` |

### Блок `detection` (режим детекции)

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `interface` | Сетевой интерфейс | `"eth0"` |
| `window_size_seconds` | Размер временного окна (сек) | `5.0` |
| `model_path` | Путь к файлу модели | `"anomaly_model.pkl"` |
| `score_threshold` | Порог по anomaly_score (`None` — бинарное предсказание модели). Чем ниже — тем выше чувствительность | `None` |
| `traffic_log_minutes` | Сколько минут трафика хранить и упаковывать в ZIP при аномалии | `5` |
| `alert_server_url` | URL для отправки архива (например, `http://127.0.0.1:5000/anomaly`) | `"http://127.0.0.1:5000/anomaly"` |
| `send_zip_on_anomaly` | Отправлять ли ZIP на веб-сервер при аномалии | `True` |

### Блок `model`

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `contamination` | Ожидаемая доля аномалий в обучающей выборке (Isolation Forest) | `0.01` |

Аргументы командной строки переопределяют соответствующие значения из конфига для текущего запуска.

## Запуск

Все команды ниже предполагают, что вы находитесь в каталоге с `main.py`, `config.py` и остальными модулями.

### 1. Режим обучения

Обучить модель на трафике с интерфейса по умолчанию и параметрами из `config.py`:

```bash
sudo python main.py --mode train
```

С указанием интерфейса, длительности и размера окна:

```bash
sudo python main.py --mode train --interface eth0 --duration 10 --window-size 5
```

Сохранение модели в отдельный файл:

```bash
sudo python main.py --mode train --model my_model.pkl --duration 5
```

Обучение должно проходить в период «нормального» трафика, без явных аномалий.

### 2. Режим детекции

Запуск детекции с моделью и параметрами по умолчанию (из `config.py`):

```bash
sudo python main.py --mode detect
```

С указанием модели, интерфейса и размера окна (размер окна должен совпадать с тем, на котором обучали):

```bash
sudo python main.py --mode detect --model my_model.pkl --interface eth0 --window-size 5
```

С заданием порога чувствительности:

```bash
sudo python main.py --mode detect --score-threshold -0.4
```

Остановка: **Ctrl+C**. В конце выведется количество обработанных окон и обнаруженных аномалий.

### 3. Демонстрация отправки архива на веб-сервер

1. В одном терминале запустите демонстрационный сервер приёма архивов:

```bash
python alert_server.py
```

Сервер слушает порт 5000 (по умолчанию `http://0.0.0.0:5000`). Убедитесь, что в `config.py` указано:
- `detection.alert_server_url = "http://127.0.0.1:5000/anomaly"`
- `detection.send_zip_on_anomaly = True` (при необходимости).

2. В другом терминале запустите детектор:

```bash
sudo python main.py --mode detect
```

При срабатывании аномалии в консоли детектора появится сообщение об отправке архива; в консоли `alert_server.py` — сообщение о полученной аномалии и путь к сохранённому ZIP в каталоге `uploads/`.

## Параметры командной строки

| Параметр | Короткая форма | Описание | По умолчанию |
|----------|----------------|----------|--------------|
| `--mode` | `-m` | Режим: `train` или `detect` | **обязательный** |
| `--interface` | `-i` | Сетевой интерфейс | из `config.py` |
| `--model` | `-M` | Путь к файлу модели | из `config.py` |
| `--duration` | `-d` | Длительность обучения (минуты), только для `train` | из `config.py` |
| `--window-size` | `-w` | Размер временного окна (секунды) | из `config.py` |
| `--score-threshold` | `-t` | Порог по anomaly_score для `detect` (ниже — выше чувствительность) | из `config.py` |

Пример полного запуска детекции:

```bash
sudo python main.py --mode detect --interface eth0 --model anomaly_model.pkl --window-size 5 --score-threshold -0.4
```

## Признаки и агрегация

### Признаки из каждого пакета

| Признак | Описание |
|---------|----------|
| `timestamp` | Время захвата |
| `src_mac`, `dst_mac` | MAC-адреса |
| `vlan`, `eth_type` | VLAN, тип Ethernet |
| `src_ip`, `dst_ip` | IP-адреса (IPv4/IPv6/ARP) |
| `proto`, `ttl` | Протокол, TTL |
| `src_port`, `dst_port` | Порты (TCP/UDP) |
| `tcp_flags` | Флаги TCP |
| `icmp_type`, `icmp_code` | ICMP |
| `length` | Длина пакета |

### Агрегация по временному окну

Пакеты группируются в окна фиксированной длины (например, 5 секунд). Для каждого окна считаются:

- число пакетов, длительность, пакетов/сек;
- уникальные src/dst IP, уникальные src/dst порты;
- счётчики по протоколам (TCP, UDP, ICMP);
- средняя/мин/макс длина пакета, средний TTL;
- количество уникальных TCP-флагов, число ICMP-пакетов.

Эти метрики образуют вектор признаков для обучения и предсказания. Размер окна задаётся в `config.py` или через `--window-size` и должен быть одинаковым при обучении и детекции.

## Настройка чувствительности

- **Через CLI:** `--score-threshold <значение>` (например, `-0.4`).
- **Через config.py:** `detection.score_threshold = -0.4` (или `None` для бинарного предсказания модели).

Для Isolation Forest:
- типичные «нормальные» score около -0.2…0.0;
- чем ниже score, тем более «аномально» образец;
- чем ниже заданный порог, тем больше срабатываний (выше чувствительность).

Рекомендуется сначала посмотреть значения score на своём трафике и затем задать порог по необходимости.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Интерфейс недоступен | Проверить список интерфейсов: `ip link show` или `ifconfig`, указать правильный `--interface` или в `config.py`. |
| Permission denied при захвате | Запускать с `sudo`. |
| Файл модели не найден | Сначала выполнить обучение: `sudo python main.py --mode train`. |
| Мало данных для обучения | Увеличить `--duration` или проверить наличие трафика на интерфейсе. |
| Архив не отправляется на сервер | Проверить, что запущен `alert_server.py`, в конфиге верный `alert_server_url` и `send_zip_on_anomaly = True`. При ошибке сети архив остаётся локально — путь выводится в консоль. |
