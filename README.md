# Детектор аномалий сетевого трафика

Программа для обучения модели без учителя на сетевом трафике и детекции аномалий в реальном времени.

## Описание

Программа использует алгоритм Isolation Forest для обучения модели без учителя на нормальном сетевом трафике, а затем детектирует аномалии в реальном времени. Поддерживаются два типа агрегации данных: по временному окну и по flow (5-tuple).

## Установка

### Требования

- Python 3.7+
- Права root (sudo) для захвата сетевых пакетов

### Установка зависимостей

```bash
pip install -r requirements.txt
```

Или вручную:

```bash
pip install scapy>=2.5.0 scikit-learn>=1.0.0 numpy>=1.21.0
```

## Использование

### Обучение модели

Обучение модели на сетевом трафике в течение N минут:

```bash
# Обучение на интерфейсе eth0 в течение 5 минут с агрегацией по временному окну
sudo python main.py --mode train --duration 5

# Обучение на другом интерфейсе в течение 10 минут
sudo python main.py --mode train --interface wlan0 --duration 10

# Обучение с агрегацией по flow (5-tuple)
sudo python main.py --mode train --aggregation flow --duration 5

# Обучение с изменением размера временного окна (10 секунд)
sudo python main.py --mode train --window-size 10 --duration 5

# Сохранение модели в другой файл
sudo python main.py --mode train --model my_model.pkl --duration 5

# Полный пример с всеми параметрами
sudo python main.py --mode train --interface eth0 --duration 10 --aggregation time_window --window-size 5 --model model.pkl
```

### Детекция аномалий

Запуск детекции аномалий с использованием сохраненной модели:

```bash
# Детекция с агрегацией по временному окну (использует модель по умолчанию)
sudo python main.py --mode detect

# Детекция с агрегацией по flow
sudo python main.py --mode detect --aggregation flow

# Использование другой модели и интерфейса
sudo python main.py --mode detect --model my_model.pkl --interface wlan0

# Изменение размера временного окна
sudo python main.py --mode detect --window-size 10

# Детекция с порогом по score (повышение чувствительности)
sudo python main.py --mode detect --score-threshold -0.3

# Полный пример
sudo python main.py --mode detect --interface eth0 --aggregation time_window --window-size 5 --model model.pkl
```

## Параметры командной строки

| Параметр | Короткая форма | Описание | По умолчанию |
|----------|---------------|----------|--------------|
| `--mode` | `-m` | Режим работы: `train` (обучение) или `detect` (детекция) | **обязательный** |
| `--interface` | `-i` | Имя сетевого интерфейса | `eth0` |
| `--model` | `-M` | Путь к файлу модели | `anomaly_model.pkl` |
| `--duration` | `-d` | Длительность обучения в минутах (только для `train`) | `5` |
| `--aggregation` | `-a` | Тип агрегации: `time_window` или `flow` | `time_window` |
| `--window-size` | `-w` | Размер временного окна в секундах (только для `time_window`) | `5.0` |
| `--score-threshold` | `-t` | Порог по anomaly_score (None = бинарное предсказание, ниже = выше чувствительность) | `None` |

## Типы агрегации

### 1. Агрегация по временному окну (`time_window`)

Пакеты группируются по фиксированным временным окнам заданного размера (по умолчанию 5 секунд). Для каждого окна вычисляются статистики:

- Количество пакетов в окне
- Пакетов в секунду
- Уникальные IP-адреса источников и назначения
- Уникальные порты источников и назначения
- Статистики по протоколам (TCP, UDP, ICMP)
- Статистики по длине пакетов (средняя, минимальная, максимальная)
- Среднее значение TTL
- Уникальные TCP флаги
- Количество ICMP пакетов

**Преимущества:**
- Фиксированные интервалы времени
- Хорошо подходит для детекции всплесков трафика
- Простая интерпретация результатов

### 2. Агрегация по flow (5-tuple) (`flow`)

Пакеты группируются по flow на основе 5-tuple:
- IP-адрес источника
- IP-адрес назначения
- Порт источника
- Порт назначения
- Протокол

Flow завершается при таймауте (по умолчанию 60 секунд без активности). Для каждого завершенного flow вычисляются статистики:

- Количество пакетов в flow
- Общий объем данных (байты)
- Длительность flow
- Пакетов и байт в секунду
- Статистики по длине пакетов
- Среднее значение TTL
- TCP флаги
- ICMP статистики

**Преимущества:**
- Анализ на уровне соединений
- Хорошо подходит для детекции подозрительных соединений
- Учет двунаправленного трафика

## Извлекаемые признаки из пакетов

Программа извлекает следующие признаки из каждого сетевого пакета:

| Признак | Уровень OSI | Описание |
|---------|-------------|----------|
| `timestamp` | - | Время захвата пакета |
| `src_mac` | 2 | MAC-адрес отправителя |
| `dst_mac` | 2 | MAC-адрес получателя |
| `vlan` | 2 | Номер VLAN |
| `eth_type` | 2 | Тип Ethernet (IPv4, ARP, IPv6 и т.д.) |
| `src_ip` | 3 | IP-адрес отправителя |
| `dst_ip` | 3 | IP-адрес получателя |
| `icmp_type` | 3 | Тип ICMP пакета |
| `icmp_code` | 3 | Код ICMP пакета |
| `proto` | - | Протокол (TCP/UDP/ICMP/другой) |
| `ttl` | 3 | Значение TTL |
| `src_port` | 4 | Порт отправителя |
| `dst_port` | 4 | Порт получателя |
| `tcp_flags` | 4 | Флаги TCP |
| `length` | - | Длина пакета |

## Алгоритм работы

1. **Обучение (`--mode train`)**:
   - Захват сетевых пакетов с указанного интерфейса в течение заданного времени
   - Извлечение features из каждого пакета
   - Агрегация пакетов (по временному окну или flow)
   - Обучение модели Isolation Forest на агрегированных данных
   - Сохранение модели в файл

2. **Детекция (`--mode detect`)**:
   - Загрузка сохраненной модели
   - Непрерывный захват пакетов в реальном времени
   - Агрегация пакетов
   - Предсказание аномалий для каждого агрегированного образца
   - Вывод информации об обнаруженных аномалиях в консоль

## Технические детали

- **Модель**: Isolation Forest из scikit-learn
- **Захват пакетов**: scapy
- **Нормализация данных**: StandardScaler
- **Сохранение модели**: pickle

## Настройка чувствительности детекции

Для повышения чувствительности обнаружения аномалий можно использовать параметр `--score-threshold`:

```bash
# Высокая чувствительность (порог -0.3)
sudo python main.py --mode detect --score-threshold -0.3

# Средняя чувствительность (порог -0.5)
sudo python main.py --mode detect --score-threshold -0.5

# Низкая чувствительность (порог -0.7)
sudo python main.py --mode detect --score-threshold -0.7

# Использование бинарного предсказания модели (по умолчанию)
sudo python main.py --mode detect
```

**Рекомендация:** Сначала обучите модель и посмотрите на значения `anomaly_score` для нормального трафика в выводе. Затем установите порог немного ниже среднего значения для повышения чувствительности.

Для Isolation Forest:
- Нормальные значения обычно в диапазоне от -0.2 до 0.0
- Аномальные значения обычно ниже -0.5
- Чем ниже порог, тем больше данных будет считаться аномальными

## Примечания

- Для захвата пакетов требуются права root (используйте `sudo`)
- Модель обучается на нормальном трафике, поэтому важно обучать на периоде без аномалий
- Размер временного окна влияет на чувствительность детекции
- Для flow агрегации используется таймаут 60 секунд для завершения flow
- Модель сохраняется в формате pickle, не передавайте недоверенные файлы моделей
- Порог по score можно установить при детекции, даже если модель была обучена без порога

## Примеры сценариев использования

### Сценарий 1: Базовое обучение и детекция

```bash
# 1. Обучить модель на 10 минут нормального трафика
sudo python main.py --mode train --duration 10

# 2. Запустить детекцию аномалий
sudo python main.py --mode detect

### Сценарий 2: Обучение и детекция с flow агрегацией

# 2. Детекция с flow агрегацией
sudo python main.py --mode detect --aggregation flow --model flow_model.pkl
```

```bash
# Обучение с окном 10 секунд

sudo python main.py --mode detect --window-size 10
```

## Устранение неполадок

**Проблема**: "Интерфейс eth0 недоступен"
- **Решение**: Проверьте доступные интерфейсы командой `ip link show` или `ifconfig`, используйте правильное имя интерфейса

**Проблема**: "Permission denied" при захвате пакетов
- **Решение**: Используйте `sudo` для запуска программы

**Проблема**: "Файл модели не найден"
- **Решение**: Сначала выполните обучение с `--mode train`

**Проблема**: Мало данных для обучения
- **Решение**: Увеличьте время обучения (`--duration`) или проверьте активность на интерфейсе
